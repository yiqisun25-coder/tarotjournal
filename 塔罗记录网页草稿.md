# 个人塔罗记录网页草稿

## 目标

做一个只给自己用的塔罗记录网页，用来保存每一次抽牌、自己的直觉解释、当时的处境，以及过一段时间后的回看感受。

这个网页不是为了“算准”，而是为了留下当时的思考痕迹。重点是：

- 记录每次占卜时真正关心的问题
- 保存自己对牌面的第一反应和完整解释
- 过一阵子回看，观察哪些理解变清楚了，哪些只是当时的情绪
- 慢慢形成自己的塔罗语言和解释习惯

## 使用场景

1. 抽牌后立刻记录
2. 几天或几周后回看
3. 搜索某张牌曾经出现过多少次
4. 对比同一个主题下，不同时间的牌面变化
5. 给每次记录补充“后来发生了什么”

## 首页结构

首页应该直接进入记录列表，不做复杂欢迎页。

建议包含：

- 顶部标题：我的塔罗记录
- 新建记录按钮
- 搜索框
- 简单筛选：
  - 全部
  - 感情
  - 工作
  - 金钱
  - 自我成长
  - 人际关系
  - 其他
- 记录列表

每条记录在列表里显示：

- 日期
- 主题分类
- 提问内容
- 使用牌阵
- 抽到的牌
- 当时情绪
- 是否已经回看

## 新建记录页面

### 基础信息

- 日期
- 时间
- 地点
- 主题分类
- 当时情绪
- 重要程度

### 提问

记录这次真正想问的问题。

示例：

- 我现在应该如何理解这段关系？
- 接下来一个月我在工作上需要注意什么？
- 我为什么一直卡在这件事上？
- 如果我选择 A，会有什么需要看见的东西？

建议增加一个字段：

- 问题背后的真实担心

因为很多时候，真正有价值的不是问题本身，而是提问时心里害怕或期待的东西。

### 牌阵

可选：

- 单张牌
- 三张牌：过去 / 现在 / 未来
- 三张牌：情况 / 阻碍 / 建议
- 五张牌
- 凯尔特十字
- 自定义牌阵

如果选择自定义牌阵，需要可以自己填写每个位置的含义。

## 每张牌的记录字段

每张牌建议记录：

- 位置
- 牌名
- 正位 / 逆位
- 第一眼感受
- 传统含义
- 我自己的解释
- 它和问题的关系
- 它和其他牌的关系
- 关键词

示例格式：

```text
位置：现状
牌名：圣杯八
方向：正位
第一眼感受：离开、失望、想去找别的东西
我的解释：我可能已经知道现在的状态不能继续给我情感满足，但还在犹豫要不要真的转身。
关键词：离开、寻找、更深的满足、承认失望
```

## 整体解释区

这里记录抽完整组牌后的总解释。

建议分成几块：

### 牌面故事

用自己的话写这组牌像是在讲什么故事。

### 给我的提醒

这次牌面最想让我看见什么。

### 我当下的理解

不要追求完美解释，先记录此刻真实的想法。

### 我准备怎么做

可以写具体行动，也可以写态度上的调整。

示例：

```text
这组牌让我感觉，我其实已经在一个不太滋养我的状态里停留很久了。
它没有直接叫我马上做决定，而是提醒我先承认自己的疲惫。
接下来我想先减少对对方反应的期待，把注意力收回到自己的生活里。
```

## 回看页面

每条记录都应该有“回看”区域。

回看可以在几天后、一周后、一个月后补充。

### 回看字段

- 回看日期
- 事情后来怎么发展
- 当时的解释哪里准
- 当时的解释哪里偏了
- 现在重新看这些牌，我会怎么理解
- 我从这次记录里学到了什么

### 回看状态

记录可以有状态：

- 未回看
- 已回看一次
- 多次回看
- 已归档

## 搜索和筛选

后续网页可以支持：

- 按关键词搜索问题和解释
- 按牌名搜索
- 按主题搜索
- 按正位 / 逆位筛选
- 按日期范围筛选
- 按情绪筛选
- 只看未回看的记录

有用的统计：

- 最常出现的牌
- 最常出现的主题
- 某张牌通常出现在哪类问题里
- 哪些牌经常一起出现
- 最近一个月的主要关键词

## 页面风格

整体感觉建议是安静、私密、适合长期回看的。

视觉方向：

- 深色或柔和浅色都可以
- 不要太花哨
- 重点是阅读舒服
- 每条记录像一页私人日志
- 牌面信息清楚，但解释文字要有足够空间

可以考虑的颜色：

- 米白
- 暗紫
- 墨绿
- 深灰
- 金色点缀

字体建议：

- 中文正文要清晰
- 标题可以稍微有一点仪式感
- 长解释区域要像日记一样好读

## 第一版功能

第一版先做最实用的：

1. 新建一条塔罗记录
2. 填写问题、牌阵、牌名和解释
3. 保存到浏览器本地
4. 首页显示所有记录
5. 可以点开查看详情
6. 可以补充回看
7. 可以搜索关键词

## 第二版功能

等第一版好用之后再加：

1. 按牌名统计出现次数
2. 按主题筛选
3. 导出 Markdown
4. 导出 JSON
5. 给每条记录加图片
6. 做月度回顾
7. 做“某张牌对我意味着什么”的个人牌义库

## 个人牌义库

后续可以单独做一个页面，记录每张牌对自己的意义。

每张牌可以包括：

- 传统关键词
- 我自己的关键词
- 我常在什么情境下抽到它
- 它给我的提醒
- 我对这张牌的情绪
- 曾经出现过的记录

示例：

```text
牌名：女祭司
我的关键词：不要急着说出来、先观察、直觉已经知道答案
常出现情境：我想问别人怎么想，但其实自己已经有感觉的时候
给我的提醒：沉默不是没有答案，有时候是不需要马上解释。
```

## 后续实现建议

可以先做成一个单页网页，不需要登录，不需要服务器，数据存在浏览器本地。

适合第一版的技术方式：

- 一个 `index.html`
- 一个 `style.css`
- 一个 `app.js`
- 使用浏览器本地存储保存记录

这样文件简单，放在电脑里直接打开就能用。等记录多了，再考虑备份、导出和更完整的数据管理。
